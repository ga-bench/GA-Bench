FIELD GUIDANCE

- "num_panels": Count of distinct visual panels or regions separated by borders, boxes, or
  whitespace. A single uninterrupted illustration = 1. Example: a figure split into three
  boxed sub-figures side by side -> 3.

- "panel_structure": Overall layout complexity.
  "simple" = 1-2 panels, easy to follow (e.g. one diagram with a caption).
  "moderate" = 3-5 panels or a clear multi-part layout.
  "complex" = many panels or densely packed sub-regions (e.g. a 6-box grid with sub-labels).

- "layout_type": The dominant spatial arrangement.
  "single_panel" = one undivided scene.
  "linear_flow" = stages laid out in a line (e.g. sample -> assay -> result).
  "cyclic_flow" = stages forming a loop (e.g. a carbon cycle).
  "grid_panels" = a matrix/grid of panels.
  "central_hub" = one central element with items radiating around it.
  "comparison" = two or more items placed for side-by-side contrast (e.g. control vs treated).
  "freeform" = scattered elements with no regular structure.

- "main_reading_direction": The path the eye is meant to follow.
  "left_to_right", "top_to_bottom", "radial" (out from a center), "circular" (around a loop),
  "mixed" (more than one direction), "none" (no implied order). Example: arrows pointing
  rightward across the image -> "left_to_right".

- "has_start_point": true if a clear visual starting element exists (e.g. an "Input" box or a
  labeled sample on the far left); otherwise false.

- "has_end_point": true if a clear visual ending element exists (e.g. a "Result"/"Conclusion"
  box or final outcome on the far right); otherwise false.

- "has_arrows_or_connectors": true if arrows, lines, or connectors link elements; otherwise
  false. Example: a flowchart with arrows -> true; a plain collage of icons -> false.

- "narrative_arc": Whether the image tells a beginning-middle-end story.
  "complete" = all three visible (e.g. problem -> method -> result).
  "partial" = only some present (e.g. method and result, but no clear problem).
  "none" = no progression, just a static scene or single concept.

- "main_entities": Every clearly distinguishable object. "name" must be transcribed text
  printed in the image OR a short visual description (do not infer identity of unlabeled
  objects). "visual_role" is its function in the figure.
  Example: [{"name": "CNN", "visual_role": "model"},
            {"name": "MRI scan", "visual_role": "input"},
            {"name": "accuracy 94%", "visual_role": "metric"}].
  Use [] if no entities are identifiable.

- "process_steps": Ordered workflow steps as short strings, in the order the image presents
  them. Example: ["collect blood sample", "extract RNA", "sequence", "analyze expression"].
  Use [] if no sequence is shown (e.g. a single static diagram).

- "relationships": Visible links between two entities, with a relationship type.
  "from"/"to" are entity names; "type" is the nature of the link.
  Examples:
    {"from": "drug", "to": "tumor", "type": "inhibition"}
    {"from": "input image", "to": "model", "type": "input_output"}
    {"from": "group A", "to": "group B", "type": "comparison"}.
  Use [] if no relationships are visible.

- "main_outcome": The single take-home message shown in the image, transcribed or
  paraphrased from visible text/result. Example: "treatment reduced tumor size by 40%".
  Use "" if no outcome is shown.

- "flow_clarity": How clearly the reading path/flow can be followed.
  "clear" = obvious arrows/order; "partially_clear" = some flow but gaps or crossings;
  "unclear" = no discernible path.

- "entity_clarity": How clearly the main entities are distinguishable and labeled.
  "clear" = each entity is distinct and labeled; "partially_clear" = some unlabeled or
  overlapping; "unclear" = entities blurred, tiny, or unidentifiable. Use "unclear" if no
  entities are identifiable.

- "relation_clarity": How clearly the relationships between entities are conveyed.
  "clear" = links are explicit and labeled; "partially_clear" = links implied but not
  explicit; "unclear" = relationships not conveyed. Use "unclear" if no relationships are
  visible.

- "main_message_identifiable": Can the GA's central point be grasped from the image alone?
  "yes" = immediately; "partially" = with effort or only in part; "no" = not identifiable.

- "visual_clutter": Density/overload of the visual.
  "low" = clean with whitespace; "medium" = fairly busy; "high" = overcrowded, hard to parse.
  Example: a packed image with many overlapping icons and text -> "high".

- "semantic_interpretability": Overall, how well the scientific meaning comes across from the

- "method_identifiable": Can the method/approach/workflow be identified from the image alone?

- "result_identifiable": Can a result/finding be identified from the image alone?

- "conclusion_identifiable": Can a conclusion/implication be identified from the image alone?
